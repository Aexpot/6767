# Исправление бага триал подписки

## Проблема
1. Триал подписка выдавалась на 1 месяц вместо 3 дней
2. Триал подписка не отображалась в профиле пользователя

## Причины
1. В схеме БД отсутствовал статус 'trial' в constraint для таблицы subscriptions
2. API `/api/vpn/links` проверял только статус 'active', игнорируя 'trial'
3. API `/api/admin/stats` и другие админ API не учитывали триал подписки

## Исправления

### 1. Обновление схемы БД
**Файл:** `migrations/add_trial_status.sql`
- Добавлен статус 'trial' в constraint таблицы subscriptions
- Добавлены поля trial_used, trial_started_at, trial_expires_at в таблицу users
- Добавлены поля sort_order, is_popular, price_per_month в таблицу subscription_plans

### 2. Исправление логики активации триала
**Файл:** `/root/x0vpn_web/app/api/trial/activate/route.ts` (строки 89-106)
- Изменен комментарий с "1 month plan" на "any plan for trial"
- Триал подписка теперь всегда создается на 3 дня независимо от плана
- Статус подписки устанавливается как 'trial'

**Файл:** `/root/x0vpn_web/lib/trial-helper.ts` (строка 33)
- Аналогичное исправление для автоактивации триала

### 3. Исправление отображения триал подписки
**Файл:** `/root/x0vpn_web/app/api/vpn/links/route.ts` (строка 20)
```sql
-- Было:
AND s.status = 'active'

-- Стало:
AND s.status IN ('active', 'trial')
```

### 4. Исправление админ панели
**Файл:** `/root/x0vpn_web/app/api/admin/stats/route.ts` (строка 20)
- Статистика активных подписок теперь включает триал подписки

**Файл:** `/root/x0vpn_web/app/api/admin/users/route.ts` (строка 59)
- Фильтр активных пользователей теперь включает триал подписки

**Файл:** `/root/x0vpn_web/app/api/admin/users/export/route.ts` (строка 28)
- Экспорт пользователей теперь включает триал подписки

## Результат
✅ Триал подписка выдается на 3 дня
✅ Триал подписка отображается в профиле пользователя
✅ Триал подписка учитывается в статистике админ панели
✅ VPN ключи доступны для пользователей с триал подпиской
